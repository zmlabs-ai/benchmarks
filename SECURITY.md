# Security Policy

ZMLabs takes the integrity of these published benchmarks — and of this repository — seriously.

## Reporting a vulnerability or a data concern
If you find a security issue, a leaked secret, or a benchmark figure that does **not** match its proof file, please report it **privately**:

- **Email:** contact.zmlabs@proton.me
- Or use GitHub's **"Report a vulnerability"** (the repository's *Security* tab → Private vulnerability reporting).

Please **do not open a public issue** for security-sensitive reports. We aim to acknowledge within a few business days.

## Scope
This repository contains **measurement files and documentation only** — there is no runnable service here. Reports about the benchmarks' accuracy or reproducibility are equally welcome (see [`AUDIT_SUMMARY.md`](AUDIT_SUMMARY.md) and [`REPRODUCIBILITY.md`](REPRODUCIBILITY.md)).
